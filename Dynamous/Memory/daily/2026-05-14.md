## [Session] Finance tracking — category recognition

### Problem
Finance entries are hard to categorize after the fact because the current input format doesn't include category information upfront.

### Options discussed
1. **Standardize input format** — prefix entries with `[CATEGORY]` so parsing is trivial, no inference needed.
2. **Build a classifier** — regex rules or lightweight ML to infer category from entry text (similar to the vault classifier explored in [[2026-05-11]]).
3. **Prompt at entry time** — interactive category picker when adding an entry, so the category is captured at the source rather than guessed later.

### Open question
Which approach fits current entry workflow? Need to know how entries are being added now (manual text, Discord command, other) before recommending one.
