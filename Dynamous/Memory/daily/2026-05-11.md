## [11:35] Phase 9 deployment

### Decisions
- Use `New-ScheduledTask*` PowerShell cmdlets over `schtasks.exe` so the heartbeat's "every 30 min, 09:00-22:00 KL" pattern can be expressed cleanly (daily trigger + copied repetition)
- Discord task: 365-day `ExecutionTimeLimit` instead of `[TimeSpan]::Zero` -- the latter is silently serialised as "kill after grace period" in PS 5.1
- Index task: daily trigger restarts a 24h repetition window every midnight, effectively forever; avoids the `[TimeSpan]::MaxValue` ISO duration range error

### Lessons
- `Register-ScheduledTask` requires an elevated PowerShell even for user-scoped tasks -- bail with a clear message rather than half-installing
- Fresh `-Daily` triggers have `$null` Repetition; you can't set `.Repetition.Interval` directly -- build a `-Once` trigger with the repetition you want and copy its `.Repetition` over
- `Unregister-ScheduledTask` kills any running instance, so re-running `install_tasks.ps1` interrupts long-running Discord bot mid-run -- expected, harmless because the wrapper restarts

### Facts
- Four `secondbrain-*` tasks registered: heartbeat (30min), reflect (08:00), index (10min), discord (logon, restart-on-failure)
- Discord wrapper logs to `.claude/data/logs/discord-YYYY-MM-DD.log` with daily rotation
- Install/uninstall scripts at `.claude/scripts/deploy/{install,uninstall}_tasks.ps1`


## [12:30] Notification channel: Toast -> Discord DM

### Decisions
- Replace Windows Toast with Discord DMs as the sole push channel -- user wants everything organised in Discord, not scattered across toast popups
- Notify path uses Discord REST API directly (urllib + Bot token), bot does NOT need to be running on the gateway for DMs to land

### Lessons
- Bot needs to share a server with the user before DMs are allowed -- 403 otherwise
- Discord's API runs behind Cloudflare; stock Python-urllib User-Agent triggers Cloudflare error 1010. Fix: set `User-Agent: DiscordBot (<url>, <version>)` per Discord docs
- Smoke-test scripts that always print "done" mislead -- the only real signal is whether stderr has `discord DM failed` or not. Worth checking the *actual* DM thread before declaring success
- `_env.py` only loads .env when explicitly imported. Heartbeat reaches `_env` transitively via `integrations` imports only if the package `__init__.py` imports it -- which it doesn't here. Heartbeat now imports `_env` directly

### Facts
- DM channel id cached at `.claude/data/discord_dm_channel.txt` so subsequent ticks skip the lookup roundtrip
- Notify falls back to stderr if DM fails -- heartbeat is never silent

## [13:30] Inbox auto-summarise + classify

### Decisions
- Two-pass design: cheap classify call (Haiku, peek the first ~3.5k chars) then full summarise call. Cleaner than one-shot structured output that risks truncation
- Lecture summary structure: key concepts -> worked examples -> study cards -> open questions
- Project summary structure (after user said "too simple"): Summary, Background, Objectives, Scope, Deliverables, Methodology, Tech stack, Key dates, Stakeholders, Risks, Notable details. Sections omitted entirely if source has no content rather than placeholder bullets
- Folder layout: `lectures/<SUBJECT>/<date>_<filename>.md` and `projects/<SUBJECT>/<Assignment_N>/<date>_<filename>.md`
- Preserve source filename casing in note filenames (`MSc_Project`, not `msc-project`) so search-by-name still works
- Deadlines auto-promoted into MEMORY.md `## Deadlines` -- imminent.scan() re-enabled so within-48h items DM you each tick

### Lessons
- Windows `print(json.dumps(..., ensure_ascii=False))` crashes on non-ASCII slide content when stdout is piped (cp1252 default). Use `sys.stdout.buffer.write(...encode("utf-8"))` instead
- Some PDFs emit Windows-1252 bytes (smart quotes, em-dashes) that aren't valid UTF-8. Subprocess capture must use `bytes` and `errors="replace"` -- not `text=True, encoding="utf-8"`
- Slug-style lowercasing for folder names is ugly and breaks search. Preserve case; only normalise whitespace and strip filesystem-forbidden chars
- Classifier must be told to prefer existing folder codes when multiple subjects appear in one doc (e.g. DIP215 vs DIP1006) -- otherwise it creates fragmented `DIP215DIP1006`-style folders

### Facts
- Classifier prompt receives an "Existing categories" list scraped from `lectures/` and `projects/` so multi-code documents pile into the established folder
- Deadline lines in MEMORY.md: `- YYYY-MM-DD — <subject> — <title>`. No HTML comment dedup key (line-content match is enough since each file processes once before moving to `_processed/`)
- Project summary prompt now ~3x more detailed than v1; omits sections rather than writing `_(none specified)_`
