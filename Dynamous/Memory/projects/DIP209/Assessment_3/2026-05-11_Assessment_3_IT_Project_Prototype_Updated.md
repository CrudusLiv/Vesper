# IT Project Prototype

## Summary
This assessment requires students to deliver an oral presentation of an IT project prototype, demonstrating successful implementation and achievement of project objectives. The presentation consists of a 10-15 minute individual presentation followed by a 5-10 minute Q&A session with the examiner. Students must explain their project design, technical implementation, system complexity, user interface design, error handling mechanisms, and testing techniques through slides and visual aids. The assessment is worth 50% of the final grade and evaluates students' ability to solve business problems through IT projects while producing relevant planning, design, implementation, and testing artifacts.

## Background / motivation
This assessment is part of DIP209/DIP2008 (IT Mini Project/Capstone Project) in Semester 3, 2024. The assignment evaluates key course learning outcomes designed to ensure students can produce functional IT projects addressing business problems and create comprehensive documentation artifacts throughout the project lifecycle.

## Objectives
The assignment assesses the following learning outcomes:
- DIP2008 – CLO1: Produce an IT project to solve business problems (C3, PLO2, MQF2)
- DIP209 – CLO2: Produce artifacts for the planning, design, implementation and testing of a software project (C3, PLO2, MQF2)

## Scope
In-scope items:
- Oral presentation (10-15 minutes duration)
- Q&A session (5-10 minutes)
- Project objectives and their relevance to problem statement
- Project requirements (comprehensive list)
- Design methodology and technical implementation details
- Modules and tools used
- Innovative features differentiating the project
- System complexity demonstration
- User interface and user experience design
- Error-free system operation and error handling mechanisms
- Testing techniques, test cases, test data, and results
- User acceptance testing (if applicable)

## Deliverables
- Presentation slides with visual aids
- Demonstrated IT project prototype
- Documentation of project objectives aligned with problem statement
- Comprehensive requirements list
- Design methodology documentation
- Technical implementation details
- System complexity analysis
- UI/UX design documentation
- Error handling mechanism documentation
- Testing documentation with test cases, data, and results

## Methodology / approach

### Part A: Presentation Content
Students must prepare slides explaining the following areas:

1. **Fulfillment of Project Objectives** (10 marks)
   - Clearly state the objectives of your project
   - Explain how each objective is relevant to the problem statement

2. **Fulfillment of Requirements** (10 marks)
   - Provide a comprehensive list of project requirements

3. **Design and Technical Implementation** (20 marks)
   - Describe the design methodology used in your project
   - Explain the technical implementation with details on modules and tools
   - Discuss any innovative features that differentiate your project from conventional solutions

4. **System Complexity, User Interface, and User Experience** (20 marks)
   - Demonstrate the complexity of the system
   - Describe the user interface, ensuring it is logical and appealing

5. **Error-Free and Error Handling** (10 marks)
   - Demonstrate that the system is developed without errors
   - Explain the error handling mechanisms implemented in the project

6. **Testing Techniques** (10 marks)
   - Outline the testing techniques used in your project
   - Provide details of test cases, test data, and results
   - Discuss the success of user acceptance testing, if applicable

### Part B: Presentation and Q&A
1. **Presentation Skills** (10 marks)
   - Organize your presentation with a clear flow of ideas
   - Exhibit confidence through posture and eye contact
   - Minimize vocal fillers and use purposeful pauses

2. **Problem-Solving Skills during Q&A** (10 marks)
   - Anticipate potential questions from the examiner
   - Understand and accurately answer questions
   - Integrate knowledge and experience to address concerns
   - Respond respectfully and thoughtfully to all questions

## Marking rubric

### Criteria (Total: 100 marks)
- Fulfillment of Project Objectives (10%)
- Fulfillment of Requirements (10%)
- Technical Implementation (20%)
- UI and UX (20%)
- Error-Free and Error Handling (10%)
- Testing Techniques (10%)
- Presentation Skills (10%)
- Problem-Solving Skills during Q&A (10%)

### Rubric levels (0–5)
- **Level 0** (Not Attempted): Missing, not demonstrated, or not presented
- **Level 1** (Poor): Incomplete/minimal implementation, unclear objectives, poor communication, significant gaps
- **Level 2** (Fair): Partial implementation, vague objectives, basic functionality, lacks polish or clarity
- **Level 3** (Good): Most requirements met, clear alignment, adequate functionality, generally organized presentation
- **Level 4** (Very Good): Comprehensive implementation, clear/specific objectives, fully functional features, well-organized with good engagement
- **Level 5** (Excellent): Exceeds expectations, exceptional clarity/specificity, high performance/stability, masterful communication and problem-solving
