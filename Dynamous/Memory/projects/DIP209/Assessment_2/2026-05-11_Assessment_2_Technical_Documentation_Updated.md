# DIP209 Technical Documentation Assignment

## Summary
DIP209/DIP2008 is an IT Mini Project or Capstone Project course assignment requiring groups of two students to produce comprehensive technical documentation of their project implementation. The assignment assesses students' ability to clearly present project artifacts and demonstrate understanding of technical concepts through documentation spanning 3,000–4,000 words. This assessment constitutes 30% of the final grade and is evaluated through a detailed rubric covering project management, design documentation, technical implementation, testing, and future development planning. Students must submit by 2:00 pm on April 7th, 2025, with no resubmission allowed and strict academic integrity requirements enforced through Turnitin plagiarism detection.

## Background / motivation
This assignment is part of the DIP209/DIP2008 IT Mini Project/Capstone Project for Semester 1, 2025, and directly assesses course learning outcomes (CLO2 for DIP2008 and CLO3 for DIP209) related to presenting project implementation artifacts and results both orally and in writing. The technical documentation requirement ensures students can communicate their software development work professionally and demonstrate mastery of the entire project lifecycle from problem analysis to testing and future enhancement planning.

## Objectives
- Present artifacts of IT project implementation, demonstrating understanding and application of technical concepts
- Evaluate clarity, completeness, and effectiveness of project documentation
- Demonstrate mastery of software development lifecycle processes
- Communicate technical work at professional documentation standards

## Scope
- In-scope: 9 required documentation sections covering problem statement, objectives, methodology, project management, analysis & design, technical implementation, testing, and future development planning
- In-scope: Weekly progress reports demonstrating continuous effort and regular supervisor engagement
- In-scope: Work Breakdown Structure (WBS) and Gantt Chart for project planning
- In-scope: Charts, diagrams, and screenshots as necessary to support documentation
- Out-of-scope: Actual software code submission (only documentation of technical implementation)

## Deliverables
- Comprehensive technical documentation report (3,000–4,000 words)
- Problem Statement with specific contextual details explaining stakeholder importance
- Project Objectives section with clear, specific, measurable, achievable goals
- Methodology/Technique/Approach section describing data collection, analysis, development, and testing
- Work Breakdown Structure (WBS) with organized task hierarchy
- Gantt Chart illustrating project timeline and task dependencies
- Analysis & Design documentation including:
  - User Requirements
  - Functional Requirements
  - Non-Functional Requirements
  - Logical Process Model
  - Logical Data Model
  - User Flow diagram
  - Hardware and Software Specifications
  - Physical Process Model
  - Physical Data Model
- Technical Implementation section outlining core modules, tools, and sources used
- Testing Strategy section with test plans, test cases, and test results
- Future Development recommendations section
- Weekly Project Progress Reports following Appendix A template
- Assignment Cover Sheet
- Proper academic referencing using APA Style (7th Edition preferred)

## Methodology / approach

### Documentation Structure and Content Requirements
The assignment requires a structured technical documentation approach covering the complete project lifecycle:

**Problem Definition**: Students must clearly articulate the problem their IT project solves and provide specific contextual details explaining why the problem is important to stakeholders.

**Objectives Definition**: Define clear, specific, and measurable objectives that are relevant to the problem statement and achievable within the project's timeframe.

**Methodology Documentation**: Describe methodology, techniques, and project approach with specific details on:
- Data collection methods
- Data analysis procedures
- Application development approach
- Application testing and evaluation methods

**Project Management**: Prepare a Work Breakdown Structure (WBS) organizing all project tasks and a Gantt Chart illustrating the project timeline with clear task dependencies and visual representation.

**Analysis & Design Documentation**: Comprehensive modeling documentation including requirements analysis (user, functional, non-functional), logical models (process and data models), user interaction flows, hardware and software specifications, and physical models.

**Technical Implementation Details**: Outline the technical implementation including core modules, specific tools used, and information sources referenced in development.

**Testing Documentation**: Provide a comprehensive testing strategy that includes test plans, documented test cases, and actual test results, with discussion of how testing ensures quality and functionality.

**Future Development Planning**: Provide recommendations for future improvements, potential enhancements, additional features, or technological advancements that could be incorporated.

**Progress Tracking**: Regular meetings with supervisor and weekly progress reports demonstrate continuous project progress and problem-solving effort.

### Marking Rubric (Total 100%)
Assessment uses 6-level marking scale (0–5) with the following criteria and weightings:

| Criterion | Weight |
|-----------|--------|
| Problem Statement | 5% |
| Project Objectives | 5% |
| Methodology/Technique/Approach | 10% |
| Project Management (WBS & Gantt Chart) | 20% |
| Analysis & Design | 20% |
| Technical Implementation | 10% |
| Testing | 10% |
| Future Development | 10% |
| Progress Report | 10% |

**Grading Levels for all criteria**:
- Level 0: Not Attempted / Missing or not presented
- Level 1: Poor — minimal detail, lacks structure, flawed
- Level 2: Fair — basic description with some detail, lacks depth
- Level 3: Good — clear and detailed with appropriate structures
- Level 4: Very Good — comprehensive with justification and minimal flaws
- Level 5: Excellent — exceptional clarity and depth with no significant flaws
